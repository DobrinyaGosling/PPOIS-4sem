Виртуальная кафедра - документация

Это веб-приложение для управления виртуальной кафедрой. Позволяет администраторам управлять студентами, преподавателями, материалами, заданиями, тестами, форумом и лекциями.

Краткое описание

Приложение состоит из двух частей: бэкенд на FastAPI и фронтэнд на HTML, CSS и JavaScript. Данные сохраняются в JSON файл между запусками.

Архитектура приложения

Приложение использует паттерн MVC:

Model - в src/model находятся сущности (Student, Teacher, Material, Assignment, Test, Forum и т.д.) и бизнес-логика в виде исключений, хранилища данных и основной класс VirtualDepartment.

View - в src/view находятся use-cases для работы с моделью. Каждый модуль содержит функции для операций над сущностями.

Controller - в src/controller находится FastAPI приложение, которое обрабатывает HTTP запросы и преобразует их в операции над моделью.

Все компоненты связаны через VirtualDepartmentService, который находится в src/view/use_cases/department.py.

Как все работает

При запуске сервер создает экземпляр FastAPI приложения, загружает данные из JSON файла в память, и становится готов обрабатывать запросы. Клиент может обращаться к API через HTTP или открыть веб-интерфейс в браузере.

Когда клиент выполняет операцию (создание студента, добавление материала и т.д.), фронтэнд отправляет запрос на API, который обрабатывает его, обновляет данные в памяти, сохраняет в файл и возвращает результат.

Установка и запуск

Требования: Python 3.13 или выше

Установка зависимостей:

python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

Запуск сервера:

python -m uvicorn src.controller.app:app --host 127.0.0.1 --port 8000

После этого приложение доступно по адресу http://127.0.0.1:8000. Swagger документация по адресу http://127.0.0.1:8000/docs.

Основной файл состояния находится в data/state.json. Путь можно изменить через переменные окружения STATE_FILE и DATA_DIR.

Структура веб-интерфейса

Главная страница загружается по корневому адресу /. На ней семь основных разделов:

Студенты - можно добавлять студентов с логином и именем, видеть список всех студентов.

Преподаватели - аналогично студентам, можно управлять преподавателями.

Материалы - можно добавлять учебные материалы с названием и содержанием.

Задания - создавать задания и видеть сданные работы с оценками.

Тесты - создавать тесты с вопросами и вариантами ответов, студенты могут проходить тесты.

Форум - создавать темы обсуждения, добавлять посты в темы.

Лекции - планировать лекции, начинать и заканчивать их, просматривать статус.

API endpoints

Все endpoints находятся под префиксом /api/v1

Студенты:

POST /api/v1/students - создать студента
GET /api/v1/students - получить список студентов

Преподаватели:

POST /api/v1/teachers - создать преподавателя
GET /api/v1/teachers - получить список преподавателей

Материалы:

POST /api/v1/materials - добавить материал
GET /api/v1/materials - получить список материалов

Задания:

POST /api/v1/assignments - создать задание
GET /api/v1/assignments - получить список заданий
POST /api/v1/submissions - отправить решение задания
GET /api/v1/submissions - получить список решений
PATCH /api/v1/submissions/{submission_id}/grade - поставить оценку за решение

Тесты:

POST /api/v1/tests - создать тест
GET /api/v1/tests - получить список тестов
POST /api/v1/tests/{test_id}/take - пройти тест
GET /api/v1/attempts - получить список попыток прохождения тестов

Форум:

POST /api/v1/threads - создать тему
GET /api/v1/threads - получить список тем
POST /api/v1/threads/{thread_id}/posts - добавить пост в тему

Лекции:

POST /api/v1/lectures - запланировать лекцию
GET /api/v1/lectures - получить список лекций
POST /api/v1/lectures/{lecture_id}/start - начать лекцию
POST /api/v1/lectures/{lecture_id}/end - закончить лекцию

Примеры запросов

Создание студента:

curl -X POST http://127.0.0.1:8000/api/v1/students \
  -H 'Content-Type: application/json' \
  -d '{"login":"ivan","name":"Ivan Ivanov"}'

Получение списка студентов:

curl http://127.0.0.1:8000/api/v1/students

Добавление материала:

curl -X POST http://127.0.0.1:8000/api/v1/materials \
  -H 'Content-Type: application/json' \
  -d '{"title":"Лекция 1","content":"Содержание материала"}'

Создание теста:

curl -X POST http://127.0.0.1:8000/api/v1/tests \
  -H 'Content-Type: application/json' \
  -d '{
    "title":"Тест 1",
    "questions":[
      {
        "prompt":"Какой ответ правильный?",
        "options":["Вариант 1","Вариант 2","Вариант 3"],
        "correct_index":1
      }
    ]
  }'

Обработка ошибок

API возвращает структурированные ошибки в формате JSON:

Валидация данных (422):

{
  "status": "error",
  "error": "request_validation_error",
  "message": "Invalid request",
  "details": {
    "errors": [...]
  }
}

Не найдено (404):

{
  "status": "error",
  "error": "not_found",
  "message": "Entity not found"
}

Внутренняя ошибка (500):

{
  "status": "error",
  "error": "internal_server_error",
  "message": "Произошла непредвиденная ошибка. Попробуйте позже."
}

Развертывание

В production рекомендуется использовать ASGI сервер gunicorn:

pip install gunicorn
gunicorn -w 4 -k uvicorn.workers.UvicornWorker src.controller.app:app

Также рекомендуется использовать обратный прокси (nginx) и HTTPS.

Для развертывания нужно:

1. Установить зависимости
2. Настроить переменные окружения (STATE_FILE, DATA_DIR и т.д.)
3. Запустить сервер через gunicorn или другой ASGI сервер
4. Настроить nginx или другой обратный прокси
5. Настроить SSL сертификаты
6. Создать systemd service или использовать Docker

Расширение функциональности

Если нужно добавить новую функцию:

1. Добавьте новую сущность в src/model/entities.py если это новый тип данных
2. Добавьте методы для работы с этой сущностью в VirtualDepartment (src/model/department.py)
3. Создайте use-case функции в src/view/use_cases/
4. Добавьте Pydantic схемы в src/controller/schemas.py
5. Добавьте API endpoints в src/controller/api/routes.py
6. Обновите HTML страницу в static/index.html и добавьте JavaScript функции в static/app.js
7. Напишите тесты для новой функции
8. Убедитесь что покрытие остается выше 95%

Обработка состояния

Состояние приложения сохраняется в JSON файле. При каждом изменении (создание студента, добавление материала и т.д.) данные сохраняются в файл. При запуске сервер загружает состояние из файла.

Это означает что данные сохраняются между перезапусками сервера. Если вы хотите очистить все данные, просто удалите файл state.json.

В production рекомендуется использовать базу данных вместо JSON файла для лучшей производительности и масштабируемости.

Ограничения

Текущая реализация имеет следующие ограничения:

1. Одноступенчатое хранилище (JSON файл) - не подходит для больших объемов данных
2. Нет аутентификации и авторизации - любой может выполнять любые операции
3. Нет истории операций - нельзя увидеть кто и когда создал студента
4. Простая валидация - некоторые проверки могут быть более строгими
5. Одноступенчатое приложение - нет разделения на микросервисы

Для production решения рекомендуется:

1. Использовать PostgreSQL или другую SQL базу данных
2. Добавить аутентификацию через JWT или OAuth
3. Добавить ролевую авторизацию (администратор, преподаватель, студент)
4. Вести логи всех операций
5. Добавить кэширование на Redis
6. Развернуть приложение в контейнерах Docker
7. Использовать Kubernetes для оркестрации

Часто задаваемые вопросы

Как изменить путь к файлу состояния?

Установите переменные окружения:

export STATE_FILE=/path/to/state.json
export DATA_DIR=/path/to/data

Где хранятся данные?

По умолчанию данные хранятся в файле data/state.json.

Как добавить аутентификацию?

Это требует изменения приложения. Нужно добавить JWT токены и middleware для проверки прав доступа. Примерно так:

1. Установить python-jose и другие зависимости
2. Добавить login endpoint который возвращает токен
3. Добавить middleware который проверяет токен перед каждым запросом
4. Обновить фронтэнд чтобы отправлять токен в заголовке Authorization

Это достаточно сложно и требует хорошего понимания FastAPI.

Как импортировать данные из другого формата?

Нужно написать скрипт который читает данные из другого источника и создает объекты через API или напрямую в памяти. Затем сохраняет состояние в JSON файл через JsonFileStorage.
